We need to supply our new factory with various 4 resources from 4 stations. For simplicity we name them as 1st, 2nd, 3rd and 4th resource (we leave it for your imagination). For this we need to calculate four routes from supply stations to the factory. But trucks will deliver supplies without stops and we can not build several layers roads. So routes should not intersected.

You are given a rectangular map divided by square cells. The map is represented as a sequence of strings, where:
- "." is a clear cell;
- "X" is an obstacle (forest, lake, etc);
- "1", "2", "3" or "4" are supply stations;
- "F" is a factory.

You should calculated four non intersected routes from each supply station to the factory. Each route is started in a station and should be ended in the factory. A route can be passed only by clear cells and should be in the map (trucks can not leave the map). Trucks can move in four directions:
- "N" north (up);
- "S" south (down);
- "E" east (right);
- "W" west (left);
A route will be represented as a string with these letters. The result should be as a sequence (list or tuple) of four routes from 1st to 4th.

For the given example (image) the result will be described as: 

["NEEEESSSWS",
 "WSSSSSSSEEENNNNEN,
 "NNNNEE",
 "NNNNWWWW"]

Input: The map as a tuple of strings.

Output: The routes as a tuple/list of strings.

Example: 

supply_routes()

How it is used: It's like to place contact roads on a circuit board. Or route traffic optimizing.

Precondition:
All test cases are solvable.
5 ≤ len(the_map) ≤ 10
all(5 ≤ len(row) ≤ 10 for row in the_map)
